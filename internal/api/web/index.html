    Settings

    Outbound Proxy

    Controls how outbound requests to downstream LLM providers are routed through HTTP proxies.

    Proxy Authentication

    Require clients to present a valid API key (Authorization: Bearer <key>) before requests are proxied. Leave empty to allow all traffic.

    Admin Password

    Password required to access the admin web UI. Enter a new password below to change it. Leave empty to keep the current password.

    Logging

    Control request logging verbosity. Info shows only gateway traffic (requests, responses, errors). Debug adds internal system messages about routing decisions, alias resolution, rule matching, and auto-translation.

    Payload Capture

    When enabled, Tresor stores the raw incoming request body and the raw downstream response body for the most recent 100 requests in memory. Click a row in the Logs tab to inspect prompts, messages, and tool calls. Disabled by default because it adds a small per-request memory cost. The capture is always taken before any plugin runs — you see the original client payload and the original downstream response, not the plugin-transformed version.
